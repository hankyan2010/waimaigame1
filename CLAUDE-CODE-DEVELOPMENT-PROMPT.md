# Claude Code 开发指令模板

## 一、项目背景
当前要开发的是一个“外卖经营知识答题挑战”项目。

这不是问卷调查，而是一个面向餐饮/外卖老板的**知识问答型互动增长工具**。

核心机制：
- 随机 10 题
- 单选题
- 有标准答案
- 答对得分
- 最终输出分数 + 段位 + 奖励
- 答对题后可选择店铺升级方向
- 店铺主画面会随着升级逐步成长为超级豪华大店

请在开始开发前，先完整阅读：
- `projects/waimai-quiz-game/PROJECT-BRIEF.md`
- `projects/waimai-quiz-game/STORE-UPGRADE-ANIMATION.md`
- `projects/waimai-quiz-game/PRD.md`

---

## 二、开发总要求
1. 不要直接跳过规划阶段
2. 每次开始实现前，先输出执行计划
3. 优先做 MVP，不要过度设计
4. 所有代码必须可维护、可扩展
5. 前台必须移动端优先
6. 前台视觉风格必须完全采用美团官方配色逻辑（黄黑白体系）
7. 后台重点是高效维护题库与活动配置
8. 完成后必须运行 lint、typecheck、test
9. 未验证通过，不得声称完成

---

## 三、技术栈建议
如果未特别指定，请优先采用：
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- PostgreSQL
- Prisma

要求：
- 全部用 TypeScript
- 不要滥用 any
- 业务逻辑不要塞进页面组件
- 数据访问尽量分层

---

## 四、第一阶段任务：先出架构，不写业务代码

请先不要实现业务代码。

先输出：
1. 项目目录结构建议
2. 前后台页面清单
3. 数据库 schema 草案
4. API 路由清单
5. 升级动画的数据结构设计
6. MVP 开发顺序
7. 风险点与规避方案

输出文件：
- `projects/waimai-quiz-game/docs/architecture.md`

---

## 五、第二阶段任务：初始化项目

初始化一个可上线的项目骨架。

要求：
1. 建立前台与后台基本目录结构
2. 配置 Tailwind
3. 配置 Prisma
4. 提供 `.env.example`
5. 提供基础 layout
6. 提供移动端友好的基础页面框架
7. 完成后运行 lint 和 typecheck

---

## 六、第三阶段任务：实现 MVP 主链路

实现以下 MVP 功能：
1. 活动首页
2. 随机抽取 10 题
3. 单选答题流程
4. 判分逻辑
5. 结果页（总分 + 段位）
6. 领奖页
7. 店铺升级主画面
8. 答对题后的升级选择
9. 升级后主画面即时变化

要求：
- 一屏一题
- 手机端优先
- 交互清晰
- 前台风格严格遵循美团官方配色逻辑

---

## 七、第四阶段任务：实现后台

实现后台最小可用版本，至少包含：
1. 活动管理
2. 题库管理
3. 题目编辑
4. 正确答案设置
5. 解析编辑
6. 分值设置
7. 段位管理
8. 参与记录查看

后台要求：
- 清晰高效
- 表单可用
- 以维护效率为优先

---

## 八、升级动画开发要求
升级动画不是简单装修，而是经营能力成长的可视化。

请确保升级系统可以支持：
- 门头升级
- 菜单升级
- 后厨升级
- 流量升级
- 口碑升级
- 会员/复购升级

每次答对题后：
- 给出 3 选 1 升级项
- 选择后即时更新主画面
- 最终形成一个完整的“超级豪华大店”结果

请把动画状态设计成可配置、可扩展的结构，不要写死成无法维护的判断地狱。

---

## 九、输出规范
每次完成任务后，请输出：
1. 执行了什么
2. 改了哪些文件
3. 如何验证
4. 当前未完成什么
5. 下一步最合理做什么

如果遇到不确定问题：
- 先分析
- 再给方案
- 不要擅自乱改方向

---

## 十、质量要求
所有实现必须满足：
- 结构清晰
- 可维护
- 可继续扩展成正式产品
- 不要为了快而堆屎山代码

在任何时候，都不要把它做成：
- 问卷调查页
- 廉价小游戏
- 土味活动页

目标始终是：

> 一个面向外卖老板、带美团官方感的知识答题挑战产品。
